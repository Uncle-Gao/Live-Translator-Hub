## Огляд проєкту

Live Translator Hub — це **настільний застосунок Electron + React GUI**, який забезпечує одночасне зріднення двох AI-інструментів програмування: Cursor та Claude. Завдяки єдиному ядру перекладу, в одному інтерфейсі керуються розгортанням рушіїв, налаштуваннями API-ключів та генерацією словників для обох цільових застосунків.

Цей проєкт є архітектурним оновленням [Live-Translator-Hub](https://github.com/Uncle-Gao/Live-Translator-Hub) — еволюція від CLI-скрипта до GUI з панеллю стану та логами в реальному часі, що об'єднує можливості зріднення Cursor та Claude в єдину платформу.


## Архітектура

```
live-translator-ecosystem/          # npm workspaces monorepo
├── packages/
│   ├── desktop-app/                # Electron + React GUI (Live Translator Hub)
│   │   ├── electron/main.js        # Головний процес, IPC-канали та збереження конфігурації
│   │   ├── electron/preload.js     # Міст зв'язку з рендер-процесом
│   │   └── src/                    # React 19 + Tailwind v4 + Zustand
│   ├── core/                       # Ядро перекладу (translator-engine.js)
│   ├── patcher-cursor/             # Патчер для застосунку Cursor
│   ├── patcher-claude/             # Патчер для застосунку Claude
│   └── dict-generator/             # AI-генератор словників
```

### Ядро перекладу

`packages/core/src/translator-engine.js` — це єдине середовище виконання, яке впроваджується в цільові застосунки. Це чистий браузерний JS без модульних залежностей. Відповідає за:

- **Зіставлення словників**: статичні записи + регулярні вирази
- **Міст AI-перекладу**: у середовищі Webview через `postMessage` надсилає запити на переклад до головного вікна, обходячи обмеження CSP
- **Кешування перекладів**: постійний кеш на основі `localStorage` з ключем `live_i18n_cache_<entity_name>`
- **Вкладений пошук у словнику**: підтримка режиму `enableNestedDict`

## Ключові можливості

### Єдине керування двома рушіями

В одному інтерфейсі керуйте статусом розгортання зріднення, версіями словників та правилами блокування для Cursor і Claude без перемикання між інструментами.

### Проникнення через Webview у всіх сценаріях

Завдяки архітектурі Translation Bridge, можливості AI-перекладу проникають з головного вікна в усі рівні плагінів Webview (наприклад, Claude Code), вирішуючи проблему блокування мережі через сувору політику CSP.

### Чотирипанельне компонування функцій

| Панель | Функція |
| :--- | :--- |
| **Cursor Engine** | Розгортання/відновлення зріднення Cursor, керування правилами блокування для головного вікна та плагінів Webview |
| **Claude Engine** | Розгортання/відновлення зріднення Claude, налаштування правил пропуску |
| **API Keys** | Керування API-ключами для кількох AI-рушіїв перекладу (підтримка OpenAI, Anthropic, Google Gemini, DeepL), ключі шифруються через Electron `safeStorage` |
| **Dict Generator** | Витяг UI-рядків із вихідного коду цільового застосунку та пакетна генерація словників перекладу через AI |

### Інтерактивне налагодження

- `Cmd + Option + Shift + B` (Mac) / `Ctrl + Alt + Shift + B` (Win) — перемикання синього пунктирного підсвічування рамок
- У режимі підсвічування утримуйте `Option` (Mac) / `Alt` (Win) і наведіть на китайський текст, щоб побачити оригінал

### Правила блокування за доменами

Кожен об'єкт (головне вікно та окремі плагіни) має повністю незалежний набір правил блокування (CSS-селектори, збіги URL, збіги заголовків), що гарантує, що область коду та основна інтерактивна зона не зазнають впливу перекладу.

### Автоматичне оновлення

Вбудований `electron-updater` підтримує автоматичну перевірку, завантаження та встановлення оновлень у macOS.

## Швидкий старт

```bash
# Встановлення залежностей
npm install

# Запуск GUI в режимі розробки
npm run dev

# Збірка дистрибутива для macOS
npm run build -w desktop-app
```

### Процес використання

1. На панелі **API Keys** налаштуйте ключі AI-рушіїв
2. Перейдіть на панель **Cursor Engine** або **Claude Engine**
3. Натисніть **Deploy** для одноразового розгортання зріднення
4. Перезапустіть цільовий застосунок, щоб зміни набули чинності

### Системні вимоги

- macOS 13+ (рекомендовано)
- Node.js 18+
- Встановлені настільні застосунки Cursor або Claude

## Безпека

- **Шифроване зберігання API-ключів**: через Electron `safeStorage` ключі зберігаються в зашифрованому вигляді в `~/.live_translator_hub/api_keys.enc`, не записуються в конфігураційні файли
- **Прямий зв'язок**: запити на переклад надсилаються безпосередньо до API AI-постачальників, без проміжних серверів
- **Ізоляція доменів**: правила блокування не торкаються вихідних файлів
