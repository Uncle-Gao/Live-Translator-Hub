## ภาพรวมโครงการ

Live Translator Hub เป็น **แอปพลิเคชันเดสก์ท็อป Electron + React GUI** ที่ให้บริการแปลภาษาแบบคลิกเดียวสำหรับเครื่องมือเขียนโปรแกรม AI สองตัวคือ Cursor และ Claude ผ่านเคอร์เนลรันไทม์การแปลแบบรวมศูนย์ จัดการการติดตั้งเอ็นจิ้น การกำหนดค่าคีย์ API และการสร้างพจนานุกรมสำหรับแอปพลิเคชันเป้าหมายทั้งสองในอินเทอร์เฟซเดียว

โปรเจกต์นี้เป็นเวอร์ชันอัปเกรดสถาปัตยกรรมของ [Live-Translator-Hub](https://github.com/Uncle-Gao/Live-Translator-Hub) — จากสคริปต์ CLI สู่ GUI ที่มีแผงสถานะและบันทึกแบบเรียลไทม์ และรวมความสามารถในการแปลภาษาของ Cursor และ Claude เป็นแพลตฟอร์มเดียวกัน

## สถาปัตยกรรม

```
live-translator-ecosystem/          # npm workspaces monorepo
├── packages/
│   ├── desktop-app/                # Electron + React GUI (Live Translator Hub)
│   │   ├── electron/main.js        # โปรเซสหลัก, ช่อง IPC และการคงค่ากำหนด
│   │   ├── electron/preload.js     # สะพานสื่อสารสำหรับโปรเซสเรนเดอร์
│   │   └── src/                    # React 19 + Tailwind v4 + Zustand
│   ├── core/                       # เคอร์เนลรันไทม์การแปล (translator-engine.js)
│   ├── patcher-cursor/             # Patcher สำหรับแอป Cursor
│   ├── patcher-claude/             # Patcher สำหรับแอป Claude
│   └── dict-generator/             # ตัวสร้างพจนานุกรม AI
```

### รันไทม์การแปล

`packages/core/src/translator-engine.js` เป็นรันไทม์เดียวที่ถูกฉีดเข้าไปในแอปพลิเคชันเป้าหมาย — เป็น JavaScript บริสุทธิ์ในเบราว์เซอร์ ไม่มีการพึ่งพาโมดูล หน้าที่ประกอบด้วย:

- **การจับคู่พจนานุกรม**: รายการคำศัพท์แบบคงที่ + รูปแบบ regex
- **สะพาน代理การแปล AI**: ในสภาพแวดล้อม Webview ใช้ `postMessage` เพื่อส่งต่อคำขอแปลไปยังหน้าต่างหลัก หลีกเลี่ยงข้อจำกัด CSP ในการเชื่อมต่อเครือข่าย
- **แคชการแปล**: แคชแบบคงอยู่บน `localStorage` โดยมีคีย์ชื่อ `live_i18n_cache_<entity_name>`
- **การค้นหาพจนานุกรมแบบซ้อน**: รองรับโหมด `enableNestedDict`

## จุดเด่นของฟังก์ชัน

### การจัดการเอ็นจิ้นคู่แบบรวมศูนย์

จัดการสถานะการติดตั้งแปลภาษา เวอร์ชันพจนานุกรม และกฎการบล็อกของ Cursor และ Claude ในอินเทอร์เฟซเดียว โดยไม่ต้องสลับเครื่องมือ

### การเจาะทะลุ Webview ทุกสถานการณ์

ผ่านสถาปัตยกรรม Translation Bridge ความสามารถในการแปล AI สามารถเจาะจากหน้าต่างหลักไปยังปลั๊กอิน Webview ทุกระดับ (เช่น Claude Code) แก้ปัญหาการบล็อกการเชื่อมต่อเครือข่ายภายใต้นโยบาย CSP ที่เข้มงวด

### เค้าโครงฟังก์ชันสี่แผง

| แผง | ฟังก์ชัน |
| :--- | :--- |
| **Cursor Engine** | ติดตั้ง/กู้คืนการแปล Cursor จัดการกฎการบล็อกแยกส่วนสำหรับหน้าต่างหลักและปลั๊กอิน Webview |
| **Claude Engine** | ติดตั้ง/กู้คืนการแปล Claude กำหนดค่ากฎการข้าม |
| **API Keys** | จัดการคีย์ API สำหรับเอ็นจิ้นการแปล AI หลายตัว (รองรับ OpenAI, Anthropic, Google Gemini, DeepL) คีย์ถูกเข้ารหัสจัดเก็บผ่าน Electron `safeStorage` |
| **Dict Generator** | ดึงสตริง UI จากซอร์สโค้ดของแอปพลิเคชันเป้าหมาย สร้างพจนานุกรมการแปลแบบกลุ่มผ่าน AI |

### การดีบักแบบโต้ตอบ

- `Cmd + Option + Shift + B` (Mac) / `Ctrl + Alt + Shift + B` (Win) สลับกรอบไฮไลท์เส้นประสีฟ้า
- ในโหมดไฮไลท์ กด `Option` (Mac) / `Alt` (Win) ค้างไว้แล้ววางเมาส์บนข้อความจีนเพื่อดูข้อความต้นฉบับ

### กฎการบล็อกแยกส่วน

แต่ละเอนทิตี (หน้าต่างหลักและปลั๊กอินต่างๆ) มีชุดกฎการบล็อกที่เป็นอิสระอย่างสมบูรณ์ (CSS selector, การจับคู่ URL, การจับคู่หัวข้อ) เพื่อให้แน่ใจว่าพื้นที่โค้ดและพื้นที่โต้ตอบหลักไม่ได้รับผลกระทบจากการแปล

### อัปเดตอัตโนมัติ

มี `electron-updater` ในตัว รองรับการตรวจสอบ ดาวน์โหลด และติดตั้งอัปเดตอัตโนมัติภายในแอปบน macOS

## เริ่มต้นใช้งานอย่างรวดเร็ว

```bash
# ติดตั้ง dependencies
npm install

# เริ่มโหมดพัฒนา GUI
npm run dev

# สร้างเวอร์ชันแจกจ่ายสำหรับ macOS
npm run build -w desktop-app
```

### ขั้นตอนการใช้งาน

1. กำหนดค่าคีย์ AI engine ในแผง **API Keys**
2. สลับไปยังแผง **Cursor Engine** หรือ **Claude Engine**
3. คลิก **Deploy** เพื่อติดตั้งการแปลแบบคลิกเดียว
4. รีสตาร์ทแอปพลิเคชันเป้าหมายเพื่อให้มีผล

### ข้อกำหนดของระบบ

- macOS 13+ (แนะนำ)
- Node.js 18+
- ติดตั้งแอปพลิเคชันเดสก์ท็อป Cursor หรือ Claude แล้ว

## ความปลอดภัย

- **การจัดเก็บคีย์ API แบบเข้ารหัส**: บันทึกผ่าน Electron `safeStorage` ไปยัง `~/.live_translator_hub/api_keys.enc` ไม่เขียนลงในไฟล์กำหนดค่า
- **การสื่อสารโดยตรง**: คำขอแปลส่งตรงไปยัง API ของผู้ให้บริการ AI ไม่มีเซิร์ฟเวอร์ตัวกลาง
- **การแยกส่วน**: กฎการบล็อกไม่แตะต้องไฟล์ซอร์สโค้ด
