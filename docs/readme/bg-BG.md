## Общ преглед на проекта

Live Translator Hub е **десктоп приложение с Electron + React GUI**, което предоставя едно кликване за китаизация на две AI инструменти за програмиране: Cursor и Claude. Чрез унифицирано ядро на преводача, то управлява внедряването на двигатели, конфигурацията на API ключове и генерирането на речници за двете целеви приложения в един интерфейс.

Този проект е архитектурна надстройка на [Live-Translator-Hub](https://github.com/Uncle-Gao/Live-Translator-Hub) – еволюция от CLI скрипт към GUI с панел за състояние и логове в реално време, обединяваща възможностите за китаизация на Cursor и Claude в една единна платформа.

## Архитектура

```
live-translator-ecosystem/          # npm workspaces monorepo
├── packages/
│   ├── desktop-app/                # Electron + React GUI (Live Translator Hub)
│   │   ├── electron/main.js        # Основен процес, IPC канали и персистентност на конфигурация
│   │   ├── electron/preload.js     # Комуникационен мост за рендиращия процес
│   │   └── src/                    # React 19 + Tailwind v4 + Zustand
│   ├── core/                       # Ядро на преводача (translator-engine.js)
│   ├── patcher-cursor/             # Patcher за приложението Cursor
│   ├── patcher-claude/             # Patcher за приложението Claude
│   └── dict-generator/             # AI генератор на речници
```

### Преводаческо ядро

`packages/core/src/translator-engine.js` е единственото ядро, инжектирано в целевите приложения – чист браузърен JS, без модулни зависимости. Отговорностите му включват:

- **Съвпадение на речници**: Статични термини + регулярни изрази
- **Мост за AI превод**: В среда на Webview използва `postMessage` за препращане на заявки за превод към главния прозорец, заобикаляйки CSP ограниченията за мрежов достъп
- **Кеш за преводи**: Персистентен кеш на база `localStorage` с ключ `live_i18n_cache_<entity_name>`
- **Търсене във вложени речници**: Поддържа режим `enableNestedDict`

## Акценти на функционалността

### Унифицирано управление на два двигателя

В един и същ интерфейс управлявайте състоянието на внедряване, версиите на речниците и правилата за блокиране за Cursor и Claude, без да сменяте инструменти.

### Проникване във всички Webview сценарии

Чрез архитектурата Translation Bridge, AI преводът може да проникне от главния прозорец до всички нива на Webview плъгини (като Claude Code), решавайки проблема с блокирането на мрежов достъп при строги CSP политики.

### Четири панела за функционалност

| Панел | Функционалност |
| :--- | :--- |
| **Cursor Engine** | Внедряване/възстановяване на китаизация за Cursor, управление на правила за блокиране за главния прозорец и Webview плъгини |
| **Claude Engine** | Внедряване/възстановяване на китаизация за Claude, конфигуриране на правила за пропускане |
| **API Keys** | Управление на API ключове за множество AI преводачи (поддържа OpenAI, Anthropic, Google Gemini, DeepL), ключовете се съхраняват криптирани чрез Electron `safeStorage` |
| **Dict Generator** | Извличане на UI низове от изходния код на целевите приложения и генериране на преводни речници чрез AI |

### Интерактивно отстраняване на грешки

- `Cmd + Option + Shift + B` (Mac) / `Ctrl + Alt + Shift + B` (Win) за превключване на синя пунктирана рамка за подчертаване
- В режим на подчертаване задръжте `Option` (Mac) / `Alt` (Win) и задръжте мишката върху китайски текст, за да видите оригиналния текст

### Правила за блокиране по домейни

Всяка единица (главен прозорец и отделни плъгини) има напълно независим набор от правила за блокиране (CSS селектори, URL съвпадения, заглавия), гарантирайки, че кодовата зона и основните интерактивни области не са засегнати от превода.

### Автоматични актуализации

Вграден `electron-updater`, поддържа автоматично проверяване, изтегляне и инсталиране на актуализации в приложението за macOS.

## Бърз старт

```bash
# Инсталиране на зависимости
npm install

# Стартиране на GUI в режим на разработка
npm run dev

# Изграждане на дистрибуция за macOS
npm run build -w desktop-app
```

### Процес на използване

1. В панела **API Keys** конфигурирайте ключовете за AI двигателите
2. Превключете към панела **Cursor Engine** или **Claude Engine**
3. Кликнете върху **Deploy** за едно кликване внедряване на китаизация
4. Рестартирайте целевото приложение, за да влезе в сила

### Системни изисквания

- macOS 13+ (препоръчително)
- Node.js 18+
- Инсталирани десктоп приложения Cursor или Claude

## Сигурност

- **Криптирано съхранение на API ключове**: Чрез Electron `safeStorage` се криптират и записват в `~/.live_translator_hub/api_keys.enc`, без да се записват в конфигурационни файлове
- **Директна комуникация**: Заявките за превод отиват директно към API на AI доставчика, без посреднически сървъри
- **Изолация по домейни**: Правилата за блокиране не докосват изходните файлове
