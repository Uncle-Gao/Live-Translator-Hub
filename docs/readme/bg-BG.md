## Общ преглед на проекта

Live Translator Hub е **десктоп приложение с Electron + React GUI**, което предоставя еднократна китаизация за два AI инструмента за програмиране – Cursor и Claude. Чрез унифицирано ядро на преводача, в един интерфейс се управлява внедряването на двигателите, конфигурацията на API ключове и генерирането на речници за двете целеви приложения.

Този проект е архитектурна надстройка на [Live-Translator-Hub](https://github.com/Uncle-Gao/Live-Translator-Hub) – еволюция от CLI скрипт към GUI с панел за състояние и логове в реално време, обединяваща възможностите за китаизация на Cursor и Claude в една обща платформа.

## Архитектура

```
live-translator-ecosystem/          # npm workspaces monorepo
├── packages/
│   ├── desktop-app/                # Electron + React GUI (Live Translator Hub)
│   │   ├── electron/main.js        # Основен процес, IPC канали и персистентност на конфигурация
│   │   ├── electron/preload.js     # Комуникационен мост за рендиращия процес
│   │   └── src/                    # React 19 + Tailwind v4 + Zustand
│   ├── core/                       # Ядро на преводача (translator-engine.js)
│   ├── patcher-cursor/             # Patcher за приложението Cursor
│   ├── patcher-claude/             # Patcher за приложението Claude
│   └── dict-generator/             # AI генератор на речници
```

### Преводаческо ядро

`packages/core/src/translator-engine.js` е единственото ядро, инжектирано в целевите приложения – чист браузърен JavaScript, без модулни зависимости. Отговорностите му включват:

- **Съвпадение на речници**: статични записи + регулярни изрази
- **Мост за AI превод**: в среда на Webview използва `postMessage` за препращане на заявки за превод към главния прозорец, заобикаляйки CSP ограниченията за мрежов достъп
- **Кеш за преводи**: персистентен кеш на база `localStorage`, с ключ `live_i18n_cache_<entity_name>`
- **Търсене във вложени речници**: поддържа режим `enableNestedDict`

## Акценти на функционалността

### Унифицирано управление на два двигателя

В един и същ интерфейс се управлява състоянието на внедряване на китаизацията за Cursor и Claude, версиите на речниците и правилата за блокиране, без да се налага превключване между инструменти.

### Проникване във всички Webview среди

Чрез архитектурата Translation Bridge, AI преводът може да проникне от главния прозорец до всички нива на Webview плъгини (като Claude Code), решавайки проблема с блокирането на мрежов достъп при строги CSP политики.

### Четири панела с функционалност

| Панел | Функционалност |
| :--- | :--- |
| **Cursor Engine** | Внедряване/възстановяване на китаизация за Cursor, управление на правила за блокиране в главния прозорец и Webview плъгини |
| **Claude Engine** | Внедряване/възстановяване на китаизация за Claude, конфигуриране на правила за пропускане |
| **API Keys** | Управление на API ключове за множество AI преводачески двигатели (поддържа OpenAI, Anthropic, Google Gemini, DeepL), ключовете се криптират чрез Electron `safeStorage` |
| **Dict Generator** | Извличане на UI низове от изходния код на целевото приложение и генериране на преводачески речници чрез AI |

### Интерактивно отстраняване на грешки

- `Cmd + Option + Shift + B` (Mac) / `Ctrl + Alt + Shift + B` (Win) – превключване на синя пунктирана рамка за подчертаване
- В режим на подчертаване задръжте `Option` (Mac) / `Alt` (Win) и задръжте курсора върху китайски текст, за да видите оригиналния текст

### Правила за блокиране по домейни

Всяка единица (главен прозорец и отделни плъгини) има напълно независим набор от правила за блокиране (CSS селектори, URL съвпадения, заглавия), гарантирайки, че кодовата зона и основните интерактивни области не се влияят от превода.

### Автоматични актуализации

Вграден `electron-updater`, поддържа автоматично проверяване, изтегляне и инсталиране на актуализации в приложението за macOS.

## Бърз старт

```bash
# Инсталиране на зависимости
npm install

# Стартиране на GUI в режим на разработка
npm run dev

# Изграждане на дистрибуция за macOS
npm run build -w desktop-app
```

### Процес на използване

1. В панела **API Keys** конфигурирайте ключовете за AI двигателя
2. Превключете към панела **Cursor Engine** или **Claude Engine**
3. Натиснете **Deploy** за еднократно внедряване на китаизацията
4. Рестартирайте целевото приложение, за да влезе в сила

### Системни изисквания

- macOS 13+ (препоръчително)
- Node.js 18+
- Инсталирани десктоп приложения Cursor или Claude

## Сигурност

- **Криптирано съхранение на API ключове**: чрез Electron `safeStorage` се записват криптирано в `~/.live_translator_hub/api_keys.enc`, не се записват в конфигурационни файлове
- **Директна комуникация**: заявките за превод отиват директно към API на AI доставчика, без междинни сървъри
- **Изолация по домейни**: правилата за блокиране не докосват изходните файлове
