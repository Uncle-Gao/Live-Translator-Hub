## परियोजना अवलोकन

Live Translator Hub एक **Electron + React GUI डेस्कटॉप एप्लिकेशन** है, जो Cursor और Claude दो AI प्रोग्रामिंग टूल्स के लिए एक-क्लिक चीनीकरण प्रदान करता है। एकीकृत अनुवाद रनटाइम कर्नेल के माध्यम से, एक इंटरफ़ेस में दो लक्ष्य अनुप्रयोगों के इंजन परिनियोजन, API कुंजी कॉन्फ़िगरेशन और शब्दकोश निर्माण का प्रबंधन करता है।

यह परियोजना [Live-Translator-Hub](https://github.com/Uncle-Gao/Live-Translator-Hub) का आर्किटेक्चरल अपग्रेड है - CLI स्क्रिप्ट से स्टेटस पैनल और रीयल-टाइम लॉग के साथ GUI में विकसित, और Cursor और Claude की चीनीकरण क्षमताओं को एक ही एकीकृत प्लेटफ़ॉर्म में विलय करता है।

## आर्किटेक्चर

```
live-translator-ecosystem/          # npm workspaces monorepo
├── packages/
│   ├── desktop-app/                # Electron + React GUI (Live Translator Hub)
│   │   ├── electron/main.js        # मुख्य प्रक्रिया, IPC चैनल और कॉन्फ़िगरेशन स्थायीकरण
│   │   ├── electron/preload.js     # रेंडर प्रक्रिया संचार ब्रिज
│   │   └── src/                    # React 19 + Tailwind v4 + Zustand
│   ├── core/                       # अनुवाद रनटाइम कर्नेल (translator-engine.js)
│   ├── patcher-cursor/             # Cursor एप्लिकेशन Patcher
│   ├── patcher-claude/             # Claude एप्लिकेशन Patcher
│   └── dict-generator/             # AI शब्दकोश जनरेटर
```

### अनुवाद रनटाइम

`packages/core/src/translator-engine.js` लक्ष्य एप्लिकेशन में इंजेक्ट किया गया एकमात्र रनटाइम है - शुद्ध ब्राउज़र JS, बिना किसी मॉड्यूल निर्भरता के। जिम्मेदारियों में शामिल हैं:

- **शब्दकोश मिलान**: स्थैतिक प्रविष्टियाँ + रेगुलर एक्सप्रेशन पैटर्न
- **AI अनुवाद प्रॉक्सी ब्रिज**: Webview वातावरण में `postMessage` के माध्यम से अनुवाद अनुरोध को मुख्य विंडो पर अग्रेषित करता है, CSP नेटवर्किंग प्रतिबंधों को बायपास करता है
- **अनुवाद कैश**: `localStorage` पर आधारित स्थायी कैश, कुंजी नाम `live_i18n_cache_<entity_name>`
- **नेस्टेड शब्दकोश खोज**: `enableNestedDict` मोड का समर्थन करता है

## विशेषताएँ

### दोहरे इंजन का एकीकृत प्रबंधन

एक ही इंटरफ़ेस में क्रमशः Cursor और Claude के चीनीकरण परिनियोजन स्थिति, शब्दकोश संस्करण और ब्लॉक नियमों का प्रबंधन करें, बिना टूल स्विच किए।

### सभी परिदृश्यों में Webview पैठ

Translation Bridge आर्किटेक्चर के माध्यम से, AI अनुवाद क्षमता मुख्य विंडो से सभी स्तरों के Webview प्लगइन्स (जैसे Claude Code) तक पहुँच सकती है, सख्त CSP नीतियों के तहत नेटवर्किंग अवरोधन समस्या को हल करती है।

### चार-पैनल फ़ंक्शन लेआउट

| पैनल | फ़ंक्शन |
| :--- | :--- |
| **Cursor Engine** | Cursor चीनीकरण परिनियोजित/पुनर्स्थापित करें, मुख्य विंडो और Webview प्लगइन्स के लिए डोमेन-विशिष्ट ब्लॉक नियम प्रबंधित करें |
| **Claude Engine** | Claude चीनीकरण परिनियोजित/पुनर्स्थापित करें, स्किप नियम कॉन्फ़िगर करें |
| **API Keys** | कई AI अनुवाद इंजनों के API कुंजियाँ प्रबंधित करें (OpenAI, Anthropic, Google Gemini, DeepL का समर्थन करता है), कुंजियाँ Electron `safeStorage` के माध्यम से एन्क्रिप्टेड संग्रहीत |
| **Dict Generator** | लक्ष्य एप्लिकेशन स्रोत कोड से UI स्ट्रिंग्स निकालें, AI के माध्यम से बैच में अनुवाद शब्दकोश उत्पन्न करें |

### इंटरैक्टिव डिबगिंग

- `Cmd + Option + Shift + B` (Mac) / `Ctrl + Alt + Shift + B` (Win) नीली डैश्ड हाइलाइट बॉर्डर टॉगल करें
- हाइलाइट मोड में `Option` (Mac) / `Alt` (Win) दबाकर रखें और चीनी टेक्स्ट पर होवर करें मूल टेक्स्ट देखने के लिए

### डोमेन-विशिष्ट ब्लॉक नियम

प्रत्येक इकाई (मुख्य विंडो और प्रत्येक प्लगइन) के पास पूरी तरह से स्वतंत्र ब्लॉक नियम सेट (CSS सेलेक्टर, URL मिलान, शीर्षक मिलान) होते हैं, यह सुनिश्चित करते हुए कि कोड क्षेत्र और मुख्य इंटरैक्शन क्षेत्र अनुवाद से प्रभावित न हों।

### स्वचालित अपडेट

अंतर्निहित `electron-updater`, macOS एप्लिकेशन के भीतर स्वचालित जाँच, डाउनलोड और अपडेट इंस्टॉलेशन का समर्थन करता है।

## त्वरित आरंभ

```bash
# निर्भरताएँ स्थापित करें
npm install

# GUI विकास मोड शुरू करें
npm run dev

# macOS वितरण योग्य संस्करण बनाएँ
npm run build -w desktop-app
```

### उपयोग प्रक्रिया

1. **API Keys** पैनल में AI इंजन कुंजियाँ कॉन्फ़िगर करें
2. **Cursor Engine** या **Claude Engine** पैनल पर स्विच करें
3. **Deploy** पर क्लिक करें एक-क्लिक चीनीकरण परिनियोजित करने के लिए
4. लक्ष्य एप्लिकेशन को पुनरारंभ करें प्रभावी होने के लिए

### सिस्टम आवश्यकताएँ

- macOS 13+ (अनुशंसित)
- Node.js 18+
- Cursor या Claude डेस्कटॉप एप्लिकेशन स्थापित

## सुरक्षा

- **API कुंजी एन्क्रिप्टेड संग्रहण**: Electron `safeStorage` के माध्यम से `~/.live_translator_hub/api_keys.enc` में एन्क्रिप्टेड सहेजा जाता है, कॉन्फ़िगरेशन फ़ाइल में नहीं लिखा जाता
- **सीधा संचार**: अनुवाद अनुरोध सीधे AI विक्रेता API तक पहुँचता है, कोई मध्यस्थ सर्वर नहीं
- **डोमेन पृथक्करण**: ब्लॉक नियम स्रोत कोड फ़ाइलों को स्पर्श नहीं करते
