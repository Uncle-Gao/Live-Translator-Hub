## Обзор проекта

**Live Translator Hub** — это **настольное приложение Electron + React с графическим интерфейсом**, которое обеспечивает русификацию в один клик для двух инструментов ИИ-программирования: Cursor и Claude. Через единое ядро времени выполнения перевода в одном интерфейсе управляется развертывание движков, настройка ключей API и генерация словарей для обоих целевых приложений.

Этот проект является архитектурным обновлением [Live-Translator-Hub](https://github.com/Uncle-Gao/Live-Translator-Hub) — эволюцией от CLI-скрипта к GUI с панелью состояния и журналом в реальном времени, объединяющим возможности русификации Cursor и Claude на единой платформе.

## Архитектура

```
live-translator-ecosystem/          # npm workspaces monorepo
├── packages/
│   ├── desktop-app/                # Electron + React GUI (Live Translator Hub)
│   │   ├── electron/main.js        # Главный процесс, IPC-каналы и сохранение конфигурации
│   │   ├── electron/preload.js     # Мост связи с процессом рендеринга
│   │   └── src/                    # React 19 + Tailwind v4 + Zustand
│   ├── core/                       # Ядро времени выполнения перевода (translator-engine.js)
│   ├── patcher-cursor/             # Патчер для приложения Cursor
│   ├── patcher-claude/             # Патчер для приложения Claude
│   └── dict-generator/             # Генератор словарей на основе ИИ
```

### Среда выполнения перевода

`packages/core/src/translator-engine.js` — это единственная среда выполнения, внедряемая в целевые приложения: чистый браузерный JS, без модульных зависимостей. Обязанности включают:

- **Сопоставление словаря**: статические записи + регулярные выражения
- **Мост-прокси для ИИ-перевода**: в среде Webview использует `postMessage` для пересылки запросов на перевод в главное окно, обходя ограничения CSP на сетевые запросы
- **Кэш переводов**: постоянный кэш на основе `localStorage` с ключом `live_i18n_cache_<entity_name>`
- **Поиск во вложенных словарях**: поддержка режима `enableNestedDict`

## Ключевые возможности

### Единое управление двумя движками

В одном интерфейсе управляйте статусом развертывания русификации, версиями словарей и правилами блокировки для Cursor и Claude без переключения между инструментами.

### Проникновение в Webview во всех сценариях

Благодаря архитектуре Translation Bridge возможности ИИ-перевода могут проникать из главного окна во все уровни плагинов Webview (например, Claude Code), решая проблему блокировки сетевых запросов при строгих политиках CSP.

### Функциональная компоновка из четырех панелей

| Панель | Функция |
| :--- | :--- |
| **Cursor Engine** | Развертывание/восстановление русификации Cursor, управление правилами блокировки для главного окна и плагинов Webview |
| **Claude Engine** | Развертывание/восстановление русификации Claude, настройка правил пропуска |
| **API Keys** | Управление ключами API для нескольких движков ИИ-перевода (поддержка OpenAI, Anthropic, Google Gemini, DeepL), ключи шифруются с помощью Electron `safeStorage` |
| **Dict Generator** | Извлечение строк интерфейса из исходного кода целевого приложения и пакетная генерация словарей перевода с помощью ИИ |

### Интерактивная отладка

- `Cmd + Option + Shift + B` (Mac) / `Ctrl + Alt + Shift + B` (Win) для переключения синей пунктирной рамки подсветки
- В режиме подсветки удерживайте `Option` (Mac) / `Alt` (Win) и наведите курсор на китайский текст, чтобы увидеть оригинал

### Правила блокировки по доменам

Каждый объект (главное окно и отдельные плагины) имеет полностью независимый набор правил блокировки (CSS-селекторы, сопоставление URL, сопоставление заголовков), что гарантирует, что области кода и основные интерактивные зоны не будут затронуты переводом.

### Автоматические обновления

Встроенный `electron-updater` поддерживает автоматическую проверку, загрузку и установку обновлений в приложении для macOS.

## Быстрый старт

```bash
# Установка зависимостей
npm install

# Запуск режима разработки GUI
npm run dev

# Сборка распространяемой версии для macOS
npm run build -w desktop-app
```

### Процесс использования

1. Настройте ключи ИИ-движка на панели **API Keys**
2. Переключитесь на панель **Cursor Engine** или **Claude Engine**
3. Нажмите **Deploy** для развертывания русификации в один клик
4. Перезапустите целевое приложение для применения изменений

### Системные требования

- macOS 13+ (рекомендуется)
- Node.js 18+
- Установленное настольное приложение Cursor или Claude

## Безопасность

- **Шифрованное хранение ключей API**: ключи шифруются с помощью Electron `safeStorage` и сохраняются в `~/.live_translator_hub/api_keys.enc`, не записываются в файлы конфигурации
- **Прямая связь**: запросы на перевод отправляются напрямую к API поставщиков ИИ, без промежуточных серверов
- **Изоляция по доменам**: правила блокировки не затрагивают исходные файлы
